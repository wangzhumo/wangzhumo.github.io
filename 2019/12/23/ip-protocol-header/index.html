IP协议头

在WPS上做了一张图,感觉会比较清楚.

MTU

Mainimum Transmission Unit 可传输的最大长度

我们现在的MTU,一般都是默认的值1500B

我们可以自己设置是否需要去分包,在Flags中可以做不分包的设置,

这样我们每一次都使用最大长度MTU传输,尽量减少分包,避免IP协议头占据了通讯的大部分

TCP协议头

  • Source Port 源端口
  • Destination Port 目标端口
  • Sequence Number 序列编号(字节数) - 可以做排序/重传/丢包啥的
  • Acknowledgment Number ACK报文,标识已经收到数据,如果不发送这个,对方重新传输
  • Data Offset 数据的开始位置
  • Reserved 保留
  • URG 紧急数据,一般不使用
  • ACK 确认报文,表示确认号是否有效
  • PSH 提示接收端这里是负载数据
  • RST 复位报文,请求对方重新建立连接
  • SYN 请求建立一个连接,3次握手
  • FIN 通知关闭连接,4次挥手
  • Window 滑动窗口,这里标识缓冲区的大小,对方一次性可以多发送一些消息
  • Checksum 校验和
  • Urgent Pointer 紧急指针
  • Options 一般使用TLV(Type-Length-Value)
  • padding 为了对齐,空数据
  • data 数据
